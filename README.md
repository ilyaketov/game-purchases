# Game Purchases

Автоматическая сборка отчётов закупа из универсальных отчётов биллинга
для последующей загрузки в QuickBooks.

Заменяет ручной процесс создания файлов вида
`2603_Продажи_на_ПЛАТИ_бс.xlsx` → лист `ЗАКУП (свод)`.

## Что внутри

- `engine.py` — движок: загрузка через calamine, классификация, агрегация, экспорт
- `config.py` — справочники площадок, поставщиков, валют (правьте здесь, а не в коде)
- `app.py` — веб-интерфейс на Streamlit с drag-and-drop и кэшированием
- `cli.py` — командная строка для скриптов и автоматизации
- `requirements.txt` — зависимости

## Производительность

На референсном железе (март 2026, ~390k строк в трёх файлах):
- Первая загрузка: ~30–45 секунд (зависит от CPU)
- Сборка одного отчёта: 0.2–2 секунды
- Сборка всех 9 отчётов: ~5 секунд после загрузки
- Повторная сборка с теми же файлами в приложении: мгновенно (кэш)

Используется `python-calamine` (Rust-движок), который в 4–5 раз быстрее стандартного openpyxl. Загружаются только нужные колонки (9 из 38 в Universal Report).

## Установка

Нужен Python 3.10+:

```bash
pip install -r requirements.txt
```

## Запуск веб-приложения

```bash
streamlit run app.py
```

Откроется http://localhost:8501. Загружаете 3 файла → выбираете площадки → жмёте кнопку → скачиваете ZIP.

При повторном выборе других площадок — загруженные файлы НЕ парсятся заново (кэш по SHA1-хэшу содержимого).

## Запуск из командной строки

Все площадки сразу:
```bash
python cli.py --r1 report1.xlsx --r2 report2.xlsx --genba genba.xlsx --out ./reports
```

Только конкретные:
```bash
python cli.py --r2 report2.xlsx --genba genba.xlsx --ploshadka Plati Eneba
```

## Логика сборки (по площадке)

1. Из R1 берутся строки с `площадка ∈ <фильтр_R1>` (1 строка = 1 ключ)
2. Из R2 берутся строки с `площадка ∈ <фильтр_R2>` (qty в колонке `Количество`)
3. Имя поставщика классифицируется в группу через `SUPPLIER_MAPPING` (config.py):
   - содержит `(Genba)` → группа `Genba`
   - содержит `(Epay)` → группа `PLN ИГРЫ. (Epay)`
   - содержит `Plug-in-Digital` или `(PID)` → группа `Plug-in-Digital`
   - иначе — снимаем префикс `XXX GAMES.` и хвост-число → ищем в словаре
4. Группируем по `(supplier_group, ID продукта)`, считаем qty
5. Цена закупа:
   - **Genba**: из genbaFile (`Grand Total / Activation Qty` по этому ID и этой площадке);
     если продукта нет в genbaFile — fallback на `Сумма в валюте базового поставщика / qty`
   - **Kishmish Games / One More Time / Callback Games** (CNY): RUB-сумма / 11
   - **Все остальные**: USD-сумма из биллинга / qty
6. Финал: группировка по поставщикам в формате `ЗАКУП (свод)` с формулами Excel

## Поддерживаемые площадки (9)

| Площадка | Источники | Покрытие |
|---|---|---|
| Plati, Kinguin, Eneba, G2A, Driffle, Tao, ChinaPlay | Биллинг + Genba | 100% автоматически |
| B2B, GamersBase | Биллинг + Genba | Только то, что есть в биллинге |

Точность по 7 полностью автоматическим площадкам — 99.93–100% от эталона.

### Особенности B2B и GamersBase

В эталонных отчётах за март 2026 эти две площадки содержат строки, которых **в биллинге нет**:

- **B2B**: ручные инвойсы PLAION (EUR, $315 840), KRM ($70 411), Giftcard pro LTD,
  СС Interactive — итого 24 940 ключей сторонних поставщиков. Genba-часть
  (2 665 ключей × $100 274.66) собирается автоматически.
- **GamersBase**: ручные инвойсы KRM, Kinguin (как сторонний поставщик); строка
  «279 ключей / $9 225» без поставщика и продукта — баг эталона (намеренно
  не воспроизводим). Биллинг-часть (83 ключа × $1 336.18) собирается автоматически.

Эти ручные строки добавляются в конец готового файла вручную, как и раньше.
Цены берутся из тех же источников, что и для остальных площадок:
genbaFile → Genba, биллинг → остальные.
